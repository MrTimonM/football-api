Football data API

Clean match, live score, event, lineup, and stat JSON.

Run a small Python server locally or on a VPS and consume football data from any frontend, dashboard, mobile app, or automation script.

View commands Deploy on VPS

Features

Live scores

Fetch active matches with normalized team, league, score, card, and corner fields.

Match details

Read timeline events, lineups, extensions, and football technical statistics.

Simple hosting

Only Python is required. Use systemd and Nginx when you are ready for a VPS.

Clean filters

Filter by sport, team, league, country, status, and refresh behavior.

Logo proxy

Team and league logos are rewritten through a local image proxy path.

Small codebase

The parser and API server are easy to inspect, edit, and deploy.

Find one match by ID.
GET

Commands And Responses

The examples below were tested against a local server running at http://127.0.0.1:8001.

Start The API

PowerShellcommand
cd e:\FootBallAPI\808fubo808
python app.py

Health Check

PowerShellcommand
Invoke-RestMethod "http://127.0.0.1:8001/api/health" |
  Select-Object ok,source,sport,matchCount,cacheTtlSeconds |
  ConvertTo-Json -Depth 6
JSONresponse
{
  "ok": true,
  "source": "https://www.808fubo808.com",
  "sport": "football",
  "matchCount": 5,
  "cacheTtlSeconds": 20
}

Find Live Matches

Search Matches

PowerShellcommand
Invoke-RestMethod "http://127.0.0.1:8001/api/matches?q=belgium" |
  ConvertTo-Json -Depth 8

Get Match Details

Use a current match ID from /api/matches. Old matches may not have detail data available.

PowerShellcommand
Invoke-RestMethod "http://127.0.0.1:8001/api/matches/{id}/details" |
  ConvertTo-Json -Depth 12

Error Response

PowerShellcommand
try {
  Invoke-RestMethod "http://127.0.0.1:8001/api/matches/not-real-id"
} catch {
  $_.ErrorDetails.Message
}
JSONresponse
{
  "error": "Match not-real-id was not found"
}

Response Shapes

Match Object

FieldTypeDescription
idstringMatch identifier.
sportstringSelected sport feed.
statusstringscheduled, live, finished, or unknown.
kickoffstring or nullISO timestamp in UTC.
leagueobjectLeague ID, name, country, and logo.
homeobjectHome team ID, name, and logo.
awayobjectAway team ID, name, and logo.
scoreobjectHome and away score.
cardsobjectRed and yellow card totals.
cornersobjectHome and away corner totals.

Detail Object

FieldTypeDescription
matchobject or nullNormalized match metadata.
hasEventsbooleanWhether event data is available.
eventsarrayGoals, cards, substitutions, penalties, and VAR events.
hasStatsbooleanWhether technical stats are available.
statsarrayParsed stat rows with code, name, home, and away values.
lineupobject or nullLineup and bench data when available.
extensionobject or nullExtra match detail data.

Filters

ParameterExampleUse
sport?sport=footballChoose football, basketball, merge, or score.
q?q=mexicoSearch league, country, team names, and slug.
league?league=premierSearch using the same behavior as q.
status?status=liveFilter by normalized match status.
refresh?refresh=trueBypass the 20-second in-memory cache.
raw?raw=trueInclude raw fields for debugging.

VPS Hosting

The recommended production-ish setup is Python behind systemd, with Nginx as a reverse proxy. Full commands are in HOSTING.md.

Ubuntuquick setup
apt update && apt upgrade -y
apt install -y python3 nginx git

git clone https://github.com/MrTimonM/football-api.git /var/www/football-api
cd /var/www/football-api
python3 -m py_compile app.py fubo808_parser.py
python3 app.py
For public domains, put Nginx in front of the Python server and use Certbot for HTTPS.
